# 多语言 (i18n) 架构说明

本篇文档介绍 NuxtShop 中的多语言 (i18n) 实现方式，包括：

- 语言包与命名空间组织方式；
- 运行时 Store 与 `useI18n` 组合式；
- SSR 与客户端语言恢复策略；
- 用户偏好与 Profile 集成；
- 如何为项目新增多语言文案。

## 1. 语言包组织

语言文件位于 `locales/` 目录：

- `locales/zh-CN.ts`：中文文案；
- `locales/en-US.ts`：英文文案。

文案按命名空间分组，常见前缀包括：

- `ui.*`：通用 UI 组件（按钮、弹窗、toast、空态等）；
- `pages.*`：各页面文案（首页、购物车、订单、个人中心等）；
- `profile.*`：个人中心相关文案；
- `demo.*`：各 Demo 页面文案（组件演示 / Pinia / HTTP / BDD 等）；
- `error.codes.*`：后端错误码的多语言提示文案。

命名规则建议保持“模块 + 场景”的结构，例如：

- `pages.cart.emptyTitle`、`pages.cart.emptyDesc`；
- `profile.security.twoFactorTitle`；
- `demo.components.modal.title`。

## 2. 运行时 Store 与 `useI18n`

### 2.1 Store：`stores/i18n.ts`

职责：

- 管理当前语言 `locale`（如 `zh-CN` / `en-US`）；
- 将用户选择持久化到 localStorage（键名 `nuxtshop-locale`）；
- 为服务端 / 客户端提供一个统一的语言读取入口。

### 2.2 组合式：`composables/useI18n.ts`

对外暴露：

- `t(key: string, params?: Record<string, any>)`：
  - 按命名空间查找文案；
  - 支持占位符，例如文案中出现 `{count}`，调用时传入 `{ count: 3 }` 即可替换；
  - 若 key 不存在，会返回原始 key，方便调试。
- `setLocale(locale: UserLanguage)`：
  - 更新 Store 中的 `locale`；
  - 写入 localStorage；
  - 可选：在登录状态下更新用户偏好（见下文）。

## 3. SSR 与客户端语言恢复

### 3.1 默认语言

- SSR 首屏使用默认语言（当前为中文 `zh-CN`），确保生成的 HTML 与客户端 Hydration 一致；
- 避免在首屏渲染时出现“服务端是中文 / 客户端变成英文”的闪烁问题。

### 3.2 客户端挂载后的恢复逻辑

客户端挂载后会按照以下优先级选择语言：

1. localStorage 中的 `nuxtshop-locale`；
2. 若无，则根据浏览器首选语言推断（如 `en` → `en-US`）；
3. 若仍无法匹配，则回退到默认语言。

恢复后的语言会写回 Pinia Store，并驱动整个应用的文案更新。

## 4. 用户偏好与 Profile 集成

### 4.1 Profile 中的语言选择

在个人中心 → 「偏好设置」中，用户可以选择：

- 简体中文 (`zh-CN`)；
- English (`en-US`)。

选择语言后会发生：

1. 调用 `setLocale` 更新当前 Store 中的语言；
2. 将语言写入 localStorage 中的 `nuxtshop-locale`；
3. 若当前为登录状态：
   - 调用 `/api/user/update`，将 `language` 字段写入用户文档；
   - 后续可以基于用户文档中的语言偏好进行 SSR 语言选择。

### 4.2 语言偏好的作用范围

- 导航与布局：顶部导航、Dashboard 左侧菜单等；
- 认证相关页面：登录 / 注册；
- 业务页面：商品列表、商品详情、购物车、订单、收藏夹、个人中心；
- 通用组件：Modal、Confirm、Loading、Pagination、两种验证码组件；
- Demo 页面：`/demos/*` 下的所有示例标题、描述和按钮。

## 5. 如何新增多语言文案

### 5.1 步骤

1. **选择命名空间**  
   根据模块选择合适的命名空间，如 `pages.checkout.*`、`profile.notifications.*`。
2. **在语言包中添加 key**  
   - 在 `locales/zh-CN.ts` 中添加中文文案；
   - 在 `locales/en-US.ts` 中添加对应英文文案；
3. **在组件中使用 `t`**  
   通过 `const { t } = useI18n()` 引入，然后：

   ```vue
   <template>
     <h1>{{ t('pages.checkout.title') }}</h1>
   </template>
   ```

4. **如有变量，使用占位符**  
   文案示例：

   ```ts
   // locales/zh-CN.ts
   pages: {
     cart: {
       itemsCount: '购物车中共有 {count} 件商品'
     }
   }
   ```

   使用：

   ```ts
   t('pages.cart.itemsCount', { count: cartCount })
   ```

### 5.2 与错误码集成

错误码使用 `error.codes.<CODE>` 命名，如：

- `error.codes.AUTH_INVALID_CREDENTIALS`
- `error.codes.USER_NOT_FOUND`

在前端，`useApiErrorHandler` 会：

- 尝试读取 `t('error.codes.' + code)`；
- 若存在则使用多语言文案；
- 否则回退到后端返回的 `message`。

## 6. 小结

NuxtShop 的 i18n 实现特点：

- 采用轻量级的自定义实现，便于阅读和按需扩展；
- 与用户偏好、错误处理、Demo 页面紧密结合；
- 对 SSR / CSR 切换友好，不容易出现 Hydration 问题。

如果你需要接入更多语言（例如日语、繁体中文），推荐做法是：

- 在 `locales/` 下新增对应语言文件（如 `ja-JP.ts`）；
- 在 `UserLanguage` 类型与 i18n Store 中补充语言枚举；
- 在 Profile 的偏好设置中增加可选项；
- 在服务器端根据用户或请求头决定初始语言。
